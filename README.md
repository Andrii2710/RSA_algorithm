# Лабораторна робота: Захищений чат на основі алгоритму RSA

Даний проєкт реалізує клієнт-серверний чат з використанням асиметричного шифрування RSA для забезпечення конфіденційності повідомлень та хешування SHA-256 для контролю їх цілісності.

## Інструкція до запуску:

Для роботи чату необхідно мати встановлений **Python**.

1. **Клонуйте репозиторій:**
   `git clone https://github.com/Andrii2710/RSA_algorithm`
   `cd RSA_algorithm`

2. **Запустіть сервер:**
   Відкрийте термінал та запустіть серверний скрипт. Він згенерує власну пару ключів та почне очікувати на підключення.
   `python server.py`

3. **Запустіть клієнтів:**
   Відкрийте два або більше нових терміналів (використовуючи Split Terminal у VS Code) та запустіть клієнтів:
   `python client.py`
   Для зміни імені користувача змініть передостанній рядок у файлі **client.py**.

## Пояснення імплементації

Проєкт складається з трьох основних модулів:

### 1. Математична основа (modular_operations.py):
Реалізовано базові алгоритми для роботи RSA:
* **Алгоритм Евкліда та зворотній алгоритм Евкліда:** Використовуються для перевірки взаємної простоти та знаходження модульного оберненого числа (пошук приватного ключа d).
* **Швидке піднесення до степеня за модулем:** Функція `pow(base, exp, mod)` для ефективного шифрування та дешифрування.
* **Автоматична генерація ключів:** Система обирає прості числа $p$ та $q$, обчислює $n$ та $\phi(n)$, а потім підбирає публічну експоненту $e$ (за замовчуванням 65537) та обчислює відповідне значення $d$.

### 2. Криптографія:
* **Конфіденційність:** Кожне повідомлення шифрується публічним ключем отримувача. Це гарантує, що лише власник відповідного приватного ключа зможе його прочитати.
* **Цілісність (Integrity):** Перед шифруванням створюється хеш повідомлення за допомогою алгоритму **SHA-256**. Хеш передається разом із зашифрованими даними у форматі JSON-пакету. Отримувач після дешифрування порівнює отриманий хеш із фактичним.
* **Обмін ключами:** При підключенні клієнт та сервер обмінюються публічними ключами у форматі JSON.

### 3. Мережева частина:
* **Socket Programming:** Використання бібліотеки socket (TCP) для стабільного з'єднання.
* **Threading:** Паралельна обробка вхідних повідомлень та введення тексту з клавіатури без блокування інтерфейсу.
* **JSON Serialization:** Всі дані (ключі, хеші, зашифровані масиви) пакуються у формат JSON для безпечної передачі через мережевий потік.

## 4. Результати запусків

* **Підключення та обмін ключами:**
  Сервер успішно реєструє клієнтів та проводить обмін публічними ключами.

* **Захищене листування:**
  Повідомлення успішно шифруються та розшифровуються.

* **Перевірка цілісності:**
  При спробі модифікації зашифрованих даних (навіть на 1 біт), система видає повідомлення про помилку цілісності: **[ Integrity Error ]: Повідомлення було підроблено!**.
